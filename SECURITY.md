# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

1. **Do NOT** open a public issue
2. Email kevin.luecke@me.com with details
3. Include steps to reproduce if possible
4. Allow time for us to respond before disclosure

## Response Timeline

- **Acknowledgment:** Within 48 hours
- **Initial assessment:** Within 1 week
- **Resolution:** Depends on severity

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.x | ✅ |

## Security Best Practices

When using this template for your startup:

1. **Never commit secrets** — Use environment variables
2. **Review access** — Limit who can push to main
3. **Update dependencies** — Keep libraries current
4. **Audit regularly** — Review access and permissions
